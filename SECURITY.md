# Security Policy

subscribers.chat is proprietary software. This document explains how to report a
security vulnerability — it is **not** an invitation for public contributions.

## Reporting a vulnerability

Report security issues **privately** to **security@subscribers.chat**.

Please do **not** open a public issue, pull request, or discussion for anything
security-related — these repositories are not the place for vulnerability details.

Where possible, include:

- a description of the issue and its impact,
- steps to reproduce (or a proof of concept),
- the affected component / repository / version,
- any suggested remediation.

## What to expect

- We aim to acknowledge reports within a few business days.
- Please allow reasonable time to investigate and ship a fix before any public
  disclosure (coordinated disclosure).

## Scope

The platform handles personal and sensitive data. Reports involving data
exposure, authentication/authorization flaws, or anything affecting user privacy
are prioritized.

Thank you for helping keep subscribers.chat and its users safe.
